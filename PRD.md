# PRD：TOEFL 选词填空 CLI

## 1. 目标
- 支持两种输入来源：本地英文文本文件、DeepSeek AI 生成文本。
- 输出题目和答案两份文件，并可额外落盘 AI 原始输入文本。
- 保持规则可配置、可维护，且不破坏现有选词策略解耦结构。

## 2. 范围
- 包含：
- CLI 运行（参数模式 + 交互模式）。
- 配置回退（`CLI > 环境变量 > config.txt > 默认值`）。
- 文件输入与 AI 输入双通道。
- 候选筛选、放宽策略、挖空渲染、文件输出。
- 不包含：
- GUI/Web。
- 多语言处理（仅英文）。
- 多供应商多协议并行支持（首版仅 DeepSeek）。
- 重试/限速/token 成本控制（后续迭代）。

## 3. 核心规则
1. 分句：按 `. ! ?`。
2. 严格模式：不从首句和倒数两句选词。
3. 候选词条件：在可挖句段内、在 `--corpus` 白名单中、长度 `>=2`、默认非专有名词、默认词元唯一。
4. 选词顺序：先固定选择 3 个高频词（在满足候选条件前提下），剩余空位再按现有流程继续选择。
5. 候选不足放宽顺序：句段限制 -> 专有名词限制 -> 词元唯一性。
6. AI 模式文本约束：英文、固定 6 句、首句总述且后续分述。
7. AI 供应商与默认策略：首版 DeepSeek，默认 `base_url=https://api.deepseek.com`，默认模型策略 `deepseek-chat`（可被 CLI/config 覆盖）。
8. AI 失败回退：API 调用失败后，进入交互式“输入本地文件路径”回退流程。
9. seed 语义：`seed` 为可选项；未提供时每次挖空结果可不同，提供时用于“同输入文本”下挖空可复现，不保证 AI 生成文本可复现。
10. 挖空样式：保留前缀 `k=max(1,floor(len*r))`（`r∈[0.4,0.6]`），其余字母用空格分隔 `_`，并编号。
11. 密钥管理：`DEEPSEEK_API_KEY` 仅从 `.env` 或系统环境变量读取，不从 `config.txt` 读取。
12. 输出文件命名：`input`、`blanks`、`answers` 输出文件名均增加时间后缀，格式为 `YYYYMMDDHHMM`，避免覆盖历史文件。
13. 文件名规范：输出文件名前缀不允许以下划线开头；若用户传入下划线前缀，系统需自动去除前导下划线（例如 `_blanks_20260310.txt` -> `blanks_20260310.txt`）。
14. 文本可读性：生成的 txt 内容按固定宽度 `80` 自动换行，避免超长单行。
15. 仓库忽略规则：项目根目录必须维护 `.gitignore`，至少覆盖 `.env`、虚拟环境目录、缓存目录等不应提交内容；同时保留 `.env.example` 可提交，便于公开仓库用户复制后填入 API key。
16. 说明文档完整性：每次规则变更后需同步更新所有说明文档，并提供项目级 `README.md` 作为统一入口。
17. 输出目录语义：当 `output` 以路径分隔符结尾（如 `ioputs/`）时，应视为输出目录，文件应写入该目录下（例如 `ioputs/blanks_YYYYMMDDHHMM.txt`）。
18. AI 主题随机性：当未显式指定 `topic` 时，AI 主题随机选择不受 `seed` 影响，避免每次生成同一篇文章；`seed` 仅用于挖空过程可复现。

## 4. 错误处理
- 文件模式下输入文件或词表文件不存在/不可读：报错，非 0 退出。
- `--blanks < 1`：报错，非 0 退出。
- DeepSeek 模式缺少 API key（`.env` 与环境变量均缺失）：报错，非 0 退出。
- AI 返回内容不满足英文/句数约束：报错，非 0 退出。
- AI 调用失败且回退路径无效：报错，非 0 退出。
- 放宽后仍无候选：报错，非 0 退出。

## 5. 验收标准
- 文件输入模式仍可按原流程生成带日期后缀的 `blanks/answers` 输出。
- AI 输入模式可生成带日期后缀的 `input/blanks/answers` 输出。
- 参数优先级符合 `CLI > 环境变量 > config.txt`。
- 未设置 `seed` 时程序可正常运行且不要求用户输入固定 seed。
- AI 失败后可以通过本地文件路径回退继续执行。
- 同一输入 + 同 seed 输出一致。
- 放宽顺序和警告行为符合规则。
- 答案保留原词大小写并可回查空号。
- `input`、`blanks`、`answers` 文件名带 `YYYYMMDDHHMM` 后缀后可重复运行且不覆盖旧文件。
- 输出文件名前缀不会以下划线开头；历史 `_xxx` 命名自动规整为 `xxx`。
- txt 输出按 `80` 列自动换行，人工阅读不再出现超长单行。
- 每次结果优先包含 3 个高频词（候选不足时按现有放宽策略退化）。
- `.gitignore` 可阻止密钥、环境与缓存文件被误提交。
- 仓库包含可提交的 `.env.example` 模板，用户复制后填写 API key 即可运行。
- `README.md` 与其他说明文档（架构、快速开始等）内容保持一致、可直接指导使用。
- `output=ioputs/` 时输出文件实际落在 `ioputs` 目录下。
- 在 AI 模式且未指定 `topic` 时，多次运行应能产生不同主题文章（不再固定）。
